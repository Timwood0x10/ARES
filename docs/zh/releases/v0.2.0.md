# ARES v0.2.0 发布说明

**发布日期：** 2026-06-12
**分支：** improve
**变更：** 119 个文件，22,634 行新增，1,302 行删除

---

## 概述

v0.2.0 引入了 Runtime 层进行 Agent 生命周期管理、Event Sourcing 实现操作恢复、支持 HITL 的动态工作流引擎、以及可插拔的 VectorStore 接口。核心理念从"Agent 自己管自己"转变为"Runtime 管 Agent"——Agent 成为可丢弃的执行器。

---

## 架构变更

### v0.1.x 架构

```
Leader Agent
├── 管理 sub-agents
├── 管理自己的生命周期
├── 管理自己的记忆
└── 单点故障
```

### v0.2.0 架构

```
Runtime 层
├── 管理所有 Agent 的生命周期
├── 自动复活失败的 Agent
├── 重放事件恢复状态
│
├── EventStore（操作恢复）
│   └── "我做到哪一步了？"
│
├── MemoryStore（认知恢复）
│   └── "我是谁？为什么这么做？"
│
└── Agents（可丢弃的执行器）
    ├── Leader Agent
    ├── Sub Agents
    └── 自定义 Agents
```

核心原则：**Agent 是可丢弃的执行器。真正的状态在 EventStore 和 MemoryStore 里。**

## 新功能

### 1. Runtime 层（`internal/runtime/`）

Agent 生命周期管理 + 自动复活。

```go
type Runtime interface {
    StartAgent(ctx, agent) error
    StopAgent(ctx, agentID) error
    RestartAgent(ctx, agentID) error
    RestoreAgent(ctx, agentID, factory) error
    NotifyAgentDead(agentID, reason)
    RegisterAgent(agent, factory)
    Start(ctx) error
    Stop() error
    Stats() RuntimeStats
}
```

- `NotifyAgentDead` — 非阻塞，触发异步复活
- `RestoreAgent` — factory → replayEvents → RestoreState → Start
- 并发关闭 + 可配置超时
- 所有 Agent goroutine 都有 panic recovery

### 2. Event Sourcing（`internal/events/`）

EventStore 接口 + 两个实现。

```go
type EventStore interface {
    Append(ctx, streamID, events, expectedVersion) error
    Read(ctx, streamID, opts) ([]*Event, error)
    ReadAll(ctx, opts) ([]*Event, error)
    Subscribe(ctx, filter) (<-chan *Event, error)
    StreamVersion(ctx, streamID) (int64, error)
}
```

- **MemoryEventStore** — 内存实现，开发测试用
- **PostgresEventStore** — PostgreSQL 实现，乐观并发控制
- 17 种事件类型，覆盖 Agent 生命周期、任务、记忆、工作流
- DLQ 自动重试 + MaxRetries

### 3. 动态工作流引擎（`internal/workflow/engine/`）

运行时可变 DAG + HITL 支持。

- **MutableDAG** — AddNode/RemoveNode/AddEdge/RemoveEdge + 增量 BFS 环检测
- **DynamicExecutor** — ApplyAtCheckpoint / ApplyImmediate 两种模式
- **GraphEventHub** — 图变更的 pub/sub 通知
- **HITL** — 步骤级 InterruptConfig、InterruptHandler、InterruptStore

### 4. 可插拔 VectorStore（`internal/storage/`）

```go
type VectorStore interface {
    Search(ctx, table, embedding, limit) ([]*SearchResult, error)
    AddEmbedding(ctx, table, id, embedding, metadata) error
    CreateCollection(ctx, name, dimension) error
}
```

- **PostgreSQL + pgvector** — 生产实现
- **In-memory** — 开发测试实现
- 用户可接入 Qdrant、Milvus、SQLite、Elasticsearch 或自定义后端

### 5. StatefulAgent 接口（`internal/agents/base/`）

```go
type StatefulAgent interface {
    RestoreState(state map[string]any) error
    ReplayEvents(events []*Event) error
    Snapshot() (map[string]any, error)
}
```

Leader 和 Sub Agent 都实现了这个接口，编译时检查。

### 6. 复活插件（`internal/plugins/resurrection/`）

通过 HealthChecker 接口实现可插拔的 Agent 复活。

- `HeartbeatAdapter` 桥接 AHP 心跳到 HealthChecker
- `Supervisor` 带并发复活去重
- 工厂模式恢复 — 每种 Agent 提供自己的工厂函数

### 7. WorkflowService API（`api/core/workflow.go`）

高层工作流编排抽象。

```go
type WorkflowService interface {
    Execute(ctx, req) (*WorkflowResponse, error)
    ExecuteStream(ctx, req) (<-chan WorkflowEvent, error)
    ListWorkflows(ctx) ([]*WorkflowSummary, error)
    GetWorkflow(ctx, id) (*WorkflowDefinition, error)
}
```

## 迁移指南

### 从 v0.1.x 迁移到 v0.2.0

1. **Runtime 可选。** 现有使用 `LeaderSupervisor` 的代码继续工作。准备好了再迁移到 `Runtime`。

2. **EventStore 可选。** 现有基于 Checkpoint 的恢复继续工作。加 EventStore 获得更丰富的恢复能力。

3. **VectorStore 是即插即用替换。** `Repository.Vector` 现在是接口。现有的 `*VectorSearcher` 自动满足接口。

4. **StatefulAgent 是可选的。** 只有想让 Agent 支持事件重放复活时才需要实现。
